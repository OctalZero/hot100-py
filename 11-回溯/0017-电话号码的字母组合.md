**总体一句话**：用回溯把每位数字对应的字母集合当成一层「选择」，依次挑一个字母拼进 `path`，拼到末尾就记录一组组合，递归回来时撤销选择，从而穷举出所有笛卡尔积。

### 🎯 面试常见追问
1. 为什么用递归回溯？
   - 问题具有"选择"的特征
   - 需要尝试所有可能性
   - 符合"树形结构"的思维模型

2. 时间复杂度是多少？
   - O(3^N × 4^M)
   - N是对应三个字母的数字个数
   - M是对应四个字母的数字个数（比如7和9）

3. 如何优化内存使用？
   - 使用StringBuilder代替String
   - 复用路径变量而不是创建新的

### 📚 解题技巧总结
记住这个"回溯三部曲"：
1. 选择：append当前字母
2. 探索：递归处理下一个数字
3. 撤销：删除当前字母

## 🐍 Python 代码

### 🥊 暴力解（朴素对照）

最朴素的多层循环思路：递归地为每一位数字挑一个字母，拼到末尾就收集——本质是手写的笛卡尔积。

```python
from typing import List

class Solution:
    def letterCombinations(self, digits: str) -> List[str]:
        if not digits:
            return []

        mapping = {
            '2': 'abc', '3': 'def', '4': 'ghi', '5': 'jkl',
            '6': 'mno', '7': 'pqrs', '8': 'tuv', '9': 'wxyz'
        }

        result = []
        path = []

        def dfs(idx: int):
            if idx == len(digits):           # 每位都挑完，记录一组组合
                result.append(''.join(path))
                return
            for ch in mapping[digits[idx]]:   # 该位数字对应的每个字母都试一遍
                path.append(ch)
                dfs(idx + 1)
                path.pop()                    # 回溯

        dfs(0)
        return result
```

- 时间复杂度：`O(3^N × 4^M)`，N/M 分别是 3/4 字母数字的个数
- 空间复杂度：`O(N+M)`，递归栈 + path
- ⚠️ 递归虽直观，但每层都要 append/pop。观察到组合数随数字增长，可直接用「滚动结果列表」迭代累乘 → 演进到下方 `O(结果数)` 的迭代解。

### ⚡ 最优解

```python
class Solution:
    def letterCombinations(self, digits: str) -> List[str]:
        if not digits:
            return []
        
        digit_to_letters = {
            '2': 'abc', '3': 'def', '4': 'ghi', '5': 'jkl',
            '6': 'mno', '7': 'pqrs', '8': 'tuv', '9': 'wxyz'
        }
        
        result = ['']
        
        for digit in digits:
            # 重新构建结果列表，每个结果加上当前数字对应的字母
            new_result = []
            for letter in digit_to_letters[digit]:
                for combination in result:
                    new_result.append(combination + letter)
            result = new_result
        
        return result
```
